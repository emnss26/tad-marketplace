5 · Flujos clave (de punta a punta)

A · Compra
Cliente→tad.com.mx (pricing)→marketplace.tad.com.mx→signup/login (magic link vía SES)→cart→checkout-session crea Subscription en PayPal→cliente aprueba en PayPal→/checkout/success→checkout-confirm escribe tenant + license en DynamoDB (idempotente por subscription_id)
B · Activación de seat
Dashboard→"Activate this PC" (hostname)→seat-activate genera token 32B→guarda SHA-256 en seats.token_hash + incrementa seats_used (atómico)→muestra plaintext UNA VEZ + install command→"Download installer" firma URL S3 de 15 min
C · Uso del MCP (runtime)
Claude Desktop→POST mcp.tad.com.mx/mcp con Authorization: Bearer + X-Client-Hostname→MCP hashea y consulta GSI token_hash_idx→valida seat.status=active + hostname + tenant.billing.status=active (cache 5 min)→encola job→add-in Revit hace long-poll /agent/poll→ejecuta en Revit→/agent/result
D · Cancelación / impago
PayPal dispara webhook→/webhooks/paypal verifica firma→license→canceled/past_due + seats→revoked/suspended + tenant.billing si aplica→el MCP rechaza esos seats en ≤5 min (TTL del cache). También: botón "Cancel subscription" del dashboard hace lo mismo + cancela en PayPal
E · Contacto comercial
tad.com.mx/contact→Web3Forms→taller.arq.dgtl@gmail.com (fallback mailto). La platform conserva /api/contact con CORS para el landing. Enterprise: mailto desde las cards de pricing
